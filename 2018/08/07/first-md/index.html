GIT的下载和安装

初始化本地仓库：git init

将本地仓库与远程仓库联系起来： ssh-keygen

配置本地环境：
git config –global user.name your gitHub name
git config –global user.email your gitHub email

建立连接：git remote add origin (origin是服务器的别名，可以更改)
更改别名：git remote rename origin dev (将origin改为dev)

GIT常使用的命令

查看远程仓库：
git remote -v

查看文件状态：
git status

将文件保存到暂存区：
git add index.html (将index.html保存到暂存区)
git add . (将所有文件保存到暂存区)

将文件放到本地仓库：
git commit -m ‘first’ (first是对内容进行描述的)

将文件提交到远程仓库:
git push origin masteer

撤销工作区的操作，在add之前的操作可以撤销：
git checkout – index.html (将工作区恢复成离自己最近一次操作的暂存区的状态)

回退操作

回退一个版本：
git reset HEAD^

回退到第几个版本：
git reset HEAD~n (n为第几个)

回退到上一个完成的版本：
git reset –hard HEAD^

hard回退到上一个工作区、暂存区、本地仓库都一致的状态

</p>

回退到上一个暂存区和本地仓库是一致的状态：
git reset –mixed HEAD^

回退到上一个工作区和暂存区一致的版本：
git reset –soft HEAD^

退回到制定的版本：
git reset –hard commitID (commitID可以通过git reflog 查看)

比较不同

比较工作区和暂存区的不同：
git diff
没有不同时，没有显示

比较暂存区和本地仓库最近一次commit后的不同：
git diff –cached

比较工作区和本地仓库最近一次commit之后的不同：
git diff HEAD

分支操作

查看分支：
git branch

创建新的分支：
git branch dev
dev是分支的名称

切换到**分支：
git checkout dev

创建并切换到新的分支：
git checkout -b dev1

合并处理：
git merge dev
在某个分支下是否merge命令时，是将dev合并到该分支中
